# 🚀 快速开始指南

## 5分钟上手 Vue Flow 流程图编辑器

### 第一步：启动项目

```bash
# 安装依赖（如果还没安装）
pnpm install

# 启动开发服务器
pnpm dev
```

### 第二步：访问页面

在浏览器中打开：`http://localhost:你的端口/test/vueflow`

### 第三步：开始使用

#### 🎯 创建你的第一个流程

1. **添加开始节点**
   - 点击底部节点面板（如果已收起）
   - 拖拽"开始"节点到画布中央

2. **添加流程节点**
   - 拖拽"流程"节点到开始节点下方
   - 在右侧属性面板中修改名称为"处理数据"

3. **连接节点**
   - 将鼠标移到开始节点的底部连接点
   - 按住鼠标左键拖动到流程节点的顶部连接点
   - 释放鼠标完成连接

4. **添加结束节点**
   - 拖拽"结束"节点到流程节点下方
   - 连接流程节点和结束节点

5. **完成！**
   - 你已经创建了一个简单的流程图
   - 点击"适应画布"按钮查看完整流程

## 📝 常用操作速查

### 节点操作

| 操作     | 方法                               |
| -------- | ---------------------------------- |
| 添加节点 | 从底部面板拖拽到画布               |
| 选择节点 | 单击节点                           |
| 移动节点 | 拖动节点                           |
| 编辑节点 | 选中后在右侧面板编辑               |
| 删除节点 | 拖到底部面板或点击属性面板删除按钮 |

### 连接操作

| 操作     | 方法                       |
| -------- | -------------------------- |
| 创建连接 | 从连接点拖动到另一个连接点 |
| 删除连接 | 点击连接线后按 Delete 键   |

### 视图操作

| 操作     | 方法                    |
| -------- | ----------------------- |
| 缩放     | 鼠标滚轮                |
| 平移     | 右键拖动或空格+左键拖动 |
| 重置视图 | 点击左上角重置按钮      |
| 适应画布 | 点击左上角适应按钮      |

## 🎨 节点类型说明

### 🟢 开始节点

- **用途**：流程的起点
- **连接**：只能输出，不能输入
- **示例**：流程开始、触发事件

### 🔵 流程节点

- **用途**：普通处理步骤
- **连接**：可输入可输出
- **示例**：数据处理、API调用、业务逻辑

### 🟠 判断节点

- **用途**：条件分支
- **连接**：一个输入，多个输出
- **示例**：if-else判断、状态检查

### ⚪ 并行节点

- **用途**：并行处理
- **连接**：一个输入，两个并行输出
- **示例**：同时执行多个任务

### 🔴 结束节点

- **用途**：流程的终点
- **连接**：只能输入，不能输出
- **示例**：流程结束、返回结果

### ⬡ 自定义节点

- **用途**：自定义功能
- **连接**：可输入可输出
- **示例**：特殊处理、扩展功能

## 💡 实用技巧

### 技巧1：快速布局

1. 先添加所有节点
2. 使用"适应画布"查看全局
3. 再调整节点位置
4. 最后创建连接

### 技巧2：批量编辑

1. 选中节点
2. 在属性面板中修改颜色
3. 为同类节点使用相同颜色便于识别

### 技巧3：使用描述

- 为关键节点添加描述说明
- 描述会显示在节点下方
- 便于团队协作理解

### 技巧4：导出数据

- 定期点击"导出数据"保存进度
- 复制控制台中的 JSON 数据
- 保存到文件作为备份

## 🎯 示例流程

### 示例1：用户注册流程

```
开始
  ↓
填写注册信息（流程节点）
  ↓
验证信息（判断节点）
  ├─ 验证通过 → 创建账户（流程节点）→ 发送欢迎邮件（流程节点）→ 结束
  └─ 验证失败 → 显示错误（流程节点）→ 结束
```

**创建步骤：**

1. 添加开始节点
2. 添加"填写注册信息"流程节点
3. 添加"验证信息"判断节点
4. 添加两个分支的流程节点
5. 添加两个结束节点
6. 依次连接各节点

### 示例2：订单处理流程

```
开始
  ↓
接收订单（流程节点）
  ↓
检查库存（判断节点）
  ├─ 有库存 → 并行处理（并行节点）
  │            ├─ 扣减库存（流程节点）
  │            └─ 生成发货单（流程节点）
  │            ↓
  │          发货（流程节点）→ 结束
  └─ 无库存 → 通知补货（流程节点）→ 结束
```

## ❓ 常见问题

### Q: 如何删除连接？

**A:** 点击连接线选中后，按键盘上的 Delete 或 Backspace 键。

### Q: 节点无法连接？

**A:** 检查节点的"可连接"属性是否开启，在右侧属性面板中可以设置。

### Q: 如何修改节点颜色？

**A:** 选中节点后，在右侧属性面板中点击颜色选择器修改。

### Q: 画布太小怎么办？

**A:** 画布会自动适应浏览器窗口，可以最大化浏览器窗口或使用缩放功能。

### Q: 如何保存流程？

**A:** 点击"导出数据"按钮，复制控制台中的 JSON 数据保存到文件。
